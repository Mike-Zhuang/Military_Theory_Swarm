# 系统设计说明

## 1. 总体目标

构建一个面向课堂演示的蜂群协同系统，覆盖从“算法生成”到“结果可视化”的完整链路：
- 仿真层：多智能体运动、协同决策、通信干扰与故障注入
- 感知层：轻量 ML 分类结果用于目标优先级调整
- 展示层：战术沙盘可视化、对照实验回放、指标看板

## 2. 模块拆分

## 2.1 sim-core

输入：仿真配置（个体数、步数、干扰参数、策略等）
输出：`scenario.json`

职责：
- 维护 Agent 与 Target 状态
- 执行离散时间步推进
- 注入通信丢包与失效事件
- 输出可回放帧序列与汇总指标

## 2.2 ml-module

输入：合成图像数据
输出：模型权重、推理置信度

职责：
- 构建可控合成数据集
- 训练轻量分类模型（Tiny CNN）
- 输出类别置信度，用于仿真任务权重调整

## 2.3 web-demo

输入：`scenario.json`
输出：可录屏网页演示

职责：
- 加载并播放帧数据
- 展示轨迹、通信拓扑、任务状态
- 显示覆盖率、完成率、响应时间等指标
- 提供对照运行（策略 A vs 策略 B）

## 4. 算法概览（教学抽象）

- 单体动力学：速度和转向受限的二维离散模型
- 协同策略：
  - 去中心化：局部邻域一致性 + Boids
  - 集中式：全局指令近似
- 任务分配：基于距离、目标优先级、ML 置信度的加权评分
- 干扰注入：随机丢包、通信半径衰减、个体失效

## 5. 非目标范围

- 不接入现实硬件控制链路
- 不提供现实部署参数
- 不实现可直接用于实战的攻击细节
